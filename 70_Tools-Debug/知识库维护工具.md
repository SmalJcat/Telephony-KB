# 知识库维护工具

知识库健康检查、Case 横向索引和 HTML 导出统一放在这里，作为维护入口。

## Case横向索引

用于从 `40_Case-Library` 的 case frontmatter 生成 [[../40_Case-Library/Case横向索引|Case横向索引]]，支持按业务域、平台、第一坏点分类、状态和 source 反查案例。脚本只统计 `doc_type: case`，不会把集合入口、外部资料片段或专题索引当成真实 case。

### 命令

```powershell
PowerShell -ExecutionPolicy Bypass -File F:\Codex\Knowledge\Telephony-KB\70_Tools-Debug\generate-case-index.ps1
```

### 适用时机

- 新增 case。
- 修改 case 的 `domain`、`platform`、`layer`、`first_bad_point`、`status`。
- 批量导入或拆分 case 后。

## 配置文档模板化

用于把 `60_Configuration` 下的配置文档统一补齐“模板化定位”区块，覆盖配置来源、匹配与生效链路、平台差异、验证证据和常见失败模式。脚本使用 `CONFIG_TEMPLATE_BLOCK_START/END` 标记，重复运行时会替换旧区块，不会继续追加重复内容。

### 命令

```powershell
PowerShell -ExecutionPolicy Bypass -File F:\Codex\Knowledge\Telephony-KB\70_Tools-Debug\normalize-config-docs.ps1
```

### 适用时机

- 新增或重构 APN、ECC、CarrierConfig、NV、运营商名称等配置文档后。
- 配置模板字段调整后，需要同步刷新各配置文档的统一入口。
- 发现配置文档只有资料摘录、缺少“怎么证明生效”时。

## 知识库健康检查

用于检查 `Telephony-KB` 是否存在断链、frontmatter 缺失、导入资料缺少阅读入口、case字段缺失、Registration case索引遗漏、LTE注册文档混入IMS主题、HTML 兼容风险和隐私残留等问题。

### 命令

```powershell
PowerShell -ExecutionPolicy Bypass -File F:\Codex\Knowledge\Telephony-KB\70_Tools-Debug\check-kb-health.ps1
```

### 当前检查项

| 检查项 | 失败条件 | 处理方式 |
|---|---|---|
| Wiki链接 | wiki链接找不到目标文件 | 修正链接或补文件 |
| Markdown本地链接 | `.md` 本地链接找不到目标文件 | 修正相对路径或改回 wiki 链接 |
| 表格内 Obsidian pipe 链接 | Markdown 表格行内使用带显示名的 wiki 链接 | 改为标准 Markdown 链接，避免表格列错位 |
| 文档 frontmatter | Markdown文件缺少文件顶部 `doc_type/domain/status/quality` | 按 [文档质量等级](../00_Index/内容归属规则.md#文档质量等级) 补齐 |
| 导入资料阅读入口 | 迁入/补充/来源记录类文档前80行没有“阅读入口/使用方法/速查结论”等入口说明 | 在顶部说明该怎么读、结论沉淀到哪里 |
| Case frontmatter | `doc_type: case` 文件没有文件顶部frontmatter，或缺少必填字段 | 按 [Case模板](../99_Templates/Case模板.md) 补齐 |
| Case status | Case `status` 不是定义过的状态值 | 按 [Case状态值](../00_Index/内容归属规则.md#Case-状态值) 修正 |
| Registration README | Registration目录下case未被README索引 | 更新 [Registration Cases](../40_Case-Library/Registration/README.md) |
| CQ隐私信息 | 文档中残留电话字段、邮箱字段、内部邮箱或紧急联系人 | 删除或脱敏 |
| CQ ID重复引用 | 同一 CQ ID 出现在多个 case | 通常保留为 warning，确认是否需要合并 |
| 大文档阅读入口 | Markdown超过默认500行且不是生成文件，也没有阅读入口 | 补阅读入口或拆到流程、排障、配置、代码、工具或case |
| LTE/Registration主题边界 | LTE注册文档或Registration case混入IMS/SIP/VoLTE关键词 | 移到 [IMS知识地图](../00_Index/业务知识地图.md#IMS) 或IMS流程/case |

### 适用时机

- 新增或迁移case后。
- 大幅整理流程文档后。
- 新增知识地图或索引页后。
- 准备打包或同步知识库前。

## HTML导出

用于把 `Telephony-KB` 的 Markdown 源文件同步为静态 HTML 阅读版。

### 命令

```powershell
python F:\Codex\Knowledge\Telephony-KB\70_Tools-Debug\export-html.py
```

默认输出目录：

```text
F:\Codex\Knowledge\Telephony-HMTL
```

### 生成内容

| 输出 | 用途 |
|---|---|
| `index.html` | HTML阅读版总入口 |
| `00_Index/Home.html` | 对应 Markdown 首页 |
| `assets/kb.css` | 全站样式 |
| `assets/kb.js` | 侧边栏搜索 |
| `search-index.json` | 页面索引 |
| `_manifest.json` | 上一次生成文件清单 |

### 维护规则

- Markdown 是唯一源文件，HTML 不手工改。
- 大幅新增或调整 Markdown 后，重新运行导出脚本。
- 如果 HTML 链接异常，先运行 [[知识库维护工具#知识库健康检查|知识库健康检查]] 确认 Markdown 侧没有断链。

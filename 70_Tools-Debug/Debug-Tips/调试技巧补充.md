## 4. Debug

If the current operator has not turned on the satellite communication switch, the following adb command can be used to set it up

1、Insert the SIM card that needs to be configured

2、Use the following adb command to read out the current operator configuration file

```java
adb pull /data/user_de/0/com.android.phone/files
```

3、Modify the current operator configuration file and add the following content to the file

```java
<boolean name="satellite_attach_supported_bool" value="true" />
<boolean name="satellite_entitlement_supported_bool" value="true" />
<boolean name="emergency_messaging_supported_bool" value="true" />
<pbundle_as_map name="carrier_supported_satellite_services_per_provider_bundle">
    <int-array name="53001" num="1">
        <item value="3"/>
    </int-array>
</pbundle_as_map>
<string name="satellite_information_redirect_url_string">https://www.entel.cl/starlink/</string>
<string name="imsserviceentitlement.entitlement_server_url_string">https://test.aes.mnc001.mcc530.pub.3gppnetwork.org:3444/rcs-entitlement/</string>
<int name="satellite_data_support_mode_int" value="0"/>
```

53001 needs to be replaced with the plmn of the current sim, and the url also needs to be configured according to the actual situation

4、Import the modified file using the following adb command

```java
adb root && adb push xxx.xml data/user_de/0/com.android.phone/files && adb shell pkill -f com.android.phone
```

xxx needs to be modified to the file name of the configuration file

## 展锐解密过程

用Wireshark工具打开ap log文件夹下的"tcpdump.cap"文件，在Wireshark工具中输入过滤关键字"isakmp"，过滤出所有的ISAKMP报文。IKE_SA_INIT MID=00表示每次重新协商IKE SA的开始，展开该报文可以看到该次IKE协商使用的Initiator SPI和Responder SPI；

 ![](../../attachments/outline/4e7a4c63-7b11-49cc-9de0-d52ccac2c106.png)

使用modem log工具对modem log进行回放

在modem log trace中搜索从ap log中获取到的Initiator SPI和Responder SPI，查找对应IKE流程；

在modem log trace中搜索PayLoadDump:SK_ai 、PPayLoadDump:SK_ar、PayLoadDump:SK_ei、PayLoadDump:SK_er，与步骤4搜索结果的时间接近，找到该次IKE协商对应的密钥以及加密算法和完整性算法。

 ![](../../attachments/outline/adac4f23-de85-4f8e-a3a0-02528c63e24a.png)

将Initiator SPI、Responder SPI、PayLoadDump:SK_ai 、PayLoadDump:SK_ar、PayLoadDump:SK_ei、PayLoadDump:SK_er以及加密算法和完整性算法按照Wireshark的IKEv2 Decryption Table格式填写，即可进行IKE消息解密。

编辑 ---》 首选项  ---》 Protocols ---> ISAKMP  --->IKEV2

 ![](../../attachments/outline/994463b2-a7e1-4548-8cad-8b41ce367ebd.png)

## 1、目的

项目认证组：FCC 法规要求，出货北美的项目，LTE B40 不能支持全频段，只能支持并测试 2305-2315 和 2350-2360MHz 这两个带宽。大部分厂家一般会选择关闭 B40, 因为测试这个频段比较难 pass (比如 bandedge fail很难整改)。针对支持 LTE B40 且要做 FCC 认证的项目，请提前确认 FCC SKU 支持的频段，建议关闭 B40。

RF: 手机支持 B40 法规要求就必须测试 B40 的 RFSPEC, 由于某种特定的原因，RF 测试一定会 Fail (没办法通过整改来 PASS)。唯一的办法就是不测试 B40, 所以只能宣称不支持，并给出不支持的机制。

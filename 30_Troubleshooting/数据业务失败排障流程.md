# 数据业务失败排障流程

## 阅读入口

这篇从“用户无法上网 / 某业务无法联网”做第一轮分诊。APN 字段和配置方法看 [APN配置方法](../60_Configuration/APN配置方法_重构.md)，数据主流程看 [Data业务流程](../20_Service-Flows/Data/Data业务流程.md)。

## 分诊顺序

| 顺序 | 判断点 | 关键证据 | 归属方向 |
|---:|---|---|---|
| 1 | 注册状态是否满足 | PS `IN_SERVICE`、RAT、roaming、data allowed | 注册/策略 |
| 2 | APN 是否选对 | APN database、`DataProfileManager`、profile id/type | 配置 |
| 3 | IA APN 是否下发 | `SET_INITIAL_ATTACH_APN`、平台 IA APN handler | AP/RIL/Modem |
| 4 | PDN 是否建立 | `SETUP_DATA_CALL`、cid、ifname、IP、DNS | Data call |
| 5 | 网络是否 reject | `PDN Connectivity Reject`、ESM cause、APN name | APN/签约/核心网 |
| 6 | 建链后是否通 | DNS、TCP、HTTP/HTTPS probe、路由 | 网络环境/系统探测 |
| 7 | 是否业务 APN | IMS/MMS/XCAP/default 区分 | 业务配置 |

## 常见分叉

| 现象 | 优先确认 |
|---|---|
| 注册 4G 但无网 | default PDN 是否成功、DNS 是否下发、NetworkMonitor 探测 |
| 只有 MMS 失败 | 是否用 MMS APN，MMSC/proxy/port 是否匹配 |
| IMS 注册失败但默认数据正常 | IMS APN、P-CSCF、IMS capability，不归入 default data |
| XCAP/补充业务失败 | XCAP APN 或复用 default、URL/AUID、GBA/BSF |
| Wi-Fi 切蜂窝异常 | 默认网络切换、data call 是否重建、validation 状态 |

## MTK 5G PDU / URSP 快速判断

MTK 5G 数据问题先确认 5GMM Registration 已完成，再看 VGSM / PDU Session。详细 MTK 关键词和补充来源看 [MTK-5G注册与PDU排障入口](MTK-5G注册与PDU排障入口.md)。

| 场景 | 第一判断 | 关键证据 |
|---|---|---|
| PDU Session Reject | 先区分 SMF 明确 reject，还是核心网没有把 N1 SM info 转发到 SMF | `VGSM_PDU_SESSION_ESTABLISHMENT_REJECT`、not forwarded cause |
| DNN / S-NSSAI 不匹配 | 常见于 slice 不支持、DNN 未签约或 URSP 选错 RSD | `DNN_NOT_SUPPORT_IN_SLICE`、`VGSM_CAUSE_MISSING_OR_UNKNOWN_DNN`、S-NSSAI |
| Emergency PDU | emergency request 不应携带普通 DNN / S-NSSAI | request type、DNN/S-NSSAI 是否为空 |
| Invalid EBI | Accept 中 EBI 合法范围应为 5-15，EBI 0 可能触发 modification | `SMIC_INVALID_MAPPED_EBI`、cause #85 |
| URSP 应用路由异常 | 应用走错 PDU / DNN / slice 时先看 URSP 规则优先级和 RSD 命中 | `AT+EGUEPOLICY`、`ursp_eval`、DNN/S-NSSAI |

## 最小证据包

- AP radio log：`DataNetworkController`、`DataProfileManager`、`RILJ`、`NetworkMonitor`。
- modem log：ESM/5GSM、PDN/PDU session、APN name、reject cause。
- 配置：目标 APN XML、运行时 APN database、`dumpsys carrier_config`。

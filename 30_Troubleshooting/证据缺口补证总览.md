# 证据缺口补证总览

## 使用入口

这篇用于处理 `summarized_with_log_gap` 和现场复现前的抓证据准备。它不替代单个 case 的补证清单；作用是先按问题域决定“下一次复现必须抓什么”，避免只拿到 AP 表象就下根因。

## 总原则

| 原则 | 要求 |
|---|---|
| 时间对齐 | AP、modem、工具截图、人工操作记录必须有同一时间基准 |
| 分层闭环 | 至少覆盖用户现象、AP 状态、RIL/vendor service、modem 协议或硬件证据 |
| 对比样本 | 能复现时优先补同卡/同地/同版本对比机；产线问题补同批正常机 |
| 配置落点 | 配置类问题必须同时保存源文件、运行时 dump 和 log 中最终生效值 |
| 第一坏点 | 结论只写第一处不满足，不把后续连锁异常当根因 |

## 注册 / 搜网 / 弱场

| 必抓证据 | 关键字段 / 关键事件 | 解决的问题 |
|---|---|---|
| AP radio/main log、bugreport | `ServiceState`、`NetworkRegistrationInfo`、`RILJ` poll/response | 判断 AP 是否正确同步 CS/PS 注册态 |
| modem NAS/RRC/L1/L2 trace | Attach/TAU Reject、RRCConnection、cell selection、RACH、measurement | 区分 NAS reject、RRC 接入失败、弱场或小区限制 |
| RF / 小区测量 | RSRP、RSRQ、SINR、RSSI、EARFCN/NRARFCN、PCI、band | 判断是否是无线条件或频段能力问题 |
| 入口动作时间线 | 开机、飞行模式、手动选网、移动路线、插卡时间 | 区分入口差异和复现步骤差异 |
| 同卡同地对比机 | 对比机同一小区和同一时间段测量值 | 防止把网络覆盖问题误判成终端问题 |

判定口径：没有 RRC/NAS trace 时，不写 Attach / TAU 根因；没有 RF 测量和对比机时，不把弱场回网问题写成终端策略缺陷。

## 数据 / APN / 默认承载

| 必抓证据 | 关键字段 / 关键事件 | 解决的问题 |
|---|---|---|
| AP data/radio log、bugreport | `SET_INITIAL_ATTACH_APN`、`setupDataCall`、DataProfile、NetworkCapabilities | 判断 APN 是否选中并下发到 modem |
| modem ESM/SM/PDN trace | PDN Connectivity Request、Activate Default EPS Bearer、ESM cause | 区分 APN 错、网络拒绝和默认承载未建 |
| APN 数据库和运行时 dump | `content://telephony/carriers`、`dumpsys telephony.registry`、`dumpsys connectivity` | 确认源配置和运行时生效值一致 |
| 网络层证据 | DNS、TCP retransmission、HTTP redirect、MMS/XCAP APN | 区分 data call 成功后的网络业务失败 |
| 复测矩阵 | 同卡不同机、不同卡同机、飞行模式恢复、清 APN 数据库 | 判断配置、订阅、缓存或网络侧策略 |

判定口径：`data call connected` 后的 DNS/TCP/MMS 失败不再归到注册；默认承载失败必须补 ESM cause 或平台 IA APN 实际下发证据。

## Call / ECC / 补充业务

| 必抓证据 | 关键字段 / 关键事件 | 解决的问题 |
|---|---|---|
| AP radio/telecom log | Dialer/Telecom、domain selection、CSFB/EPSFB request、call state | 判断拨号入口和域选是否正确 |
| modem NAS/RRC/CC/SIP 相关 trace | Extended Service Request、SRVCC、CS call control、Q.850、SIP response | 区分 CS、IMS、SRVCC 和网络释放 |
| ECC 配置和运行时号码池 | EF_ECC、database、EmergencyNumberTracker、URN/category | 判断紧急号码识别和 fallback 是否符合需求 |
| 补充业务 HTTP/AT 证据 | XCAP URL、HTTP status、BSF/GBA、AT+CCFC/ECUSD、CISS | 定位 CF/CB/USSD 是配置、认证还是 CS 回落问题 |
| 对比机和签约信息 | 同卡对比、运营商开通状态、网络返回码 | 防止把签约/网络策略误判为终端实现 |

判定口径：SRVCC/CSFB 要找切换前最后一条正常语音承载证据和切换后第一条释放证据；补充业务要先确认目标域，再查 XCAP/CS/SIP。

## IMS / VoWiFi / ViLTE

| 必抓证据 | 关键字段 / 关键事件 | 解决的问题 |
|---|---|---|
| AP IMS、radio、bugreport | IMS registration、feature capability、MMTel state、crash/tombstone | 判断 AP 能力、UI、进程崩溃和状态同步 |
| modem IMS/SIP/IWLAN trace | SIP REGISTER/INVITE、401/403/488、IKE/ePDG、P-CSCF、RTP | 区分注册、鉴权、域选、媒体承载问题 |
| IMS / CarrierConfig / SBP dump | `dumpsys carrier_config`、IMS profile、SBP/DSBP/CXP、VoWiFi 开关 | 确认运营商定制是否生效 |
| 媒体证据 | QCI1/QCI2、RTP、分辨率协商、video codec、surface/render log | 区分 modem 媒体流、AP media/render、UI 问题 |
| Wi-Fi / ePDG 证据 | Wi-Fi RSSI、DNS、IKE 算法、NAT、证书、重连时序 | 判断 VoWiFi 是接入网、IKE 还是 IMS 层失败 |

判定口径：视频闪退/卡死没有 AP crash、perfetto 或 media log 时，不把根因写到 modem；VoWiFi 没有 IKE/ePDG 证据时，不写成 SIP 根因。

## SIM / 运营商名 / UICC

| 必抓证据 | 关键字段 / 关键事件 | 解决的问题 |
|---|---|---|
| AP UICC/radio log | SIM state、UiccCard、SubscriptionInfo、ICCID/IMSI/EF 读取 | 判断 AP 是否收到卡状态和 EF 数据 |
| modem SIM trace | ATR、PPS、APDU、SIM hotplug、VSIM、slot mapping | 区分 No ATR、无中断、卡槽映射和 EF 读取失败 |
| 硬件证据 | VSIM/VIO 供电、CLK/RST/IO 波形、卡座中断、BOM/原理图 | 判断是否为硬件或驱动电源问题 |
| EF 文件和配置 | EF_SPN、EF_PNN/OPL、EF_ECC、numeric_operator、CarrierConfig | 定位名称、ECC、漫游图标和 MVNO 显示 |
| 交叉验证 | 同卡不同机、不同卡同机、单双卡槽互换、冷/热插拔 | 区分卡、卡槽、软件版本和硬件个体 |

判定口径：只有 AP 显示“无 SIM”不能直接归因 SIM 卡；必须补 ATR/中断/供电或 slot mapping 证据。

## NV / 产物 / Stability

| 必抓证据 | 关键字段 / 关键事件 | 解决的问题 |
|---|---|---|
| modem assert / EE / sysdump | assert file、line、para、modem build time、exception stack | 判断是否为 modem/NV/RF 参数第一坏点 |
| NV/NVRAM/fixnv 回读 | LID size、verno、SML、fixnv 主备、NVDATA、RF calibration | 判断个体化参数是否丢失、错版本或解密失败 |
| 版本和产物链路 | PAC、modem image、patch list、CR list、流水线参数、刷机工具日志 | 判断是否刷错 modem、漏带 patch 或产物不一致 |
| 产线/售后步骤 | 写 IMEI、校准、META、ResearchDownload/FactoryDownload 操作记录 | 还原参数在哪个环节被覆盖或写坏 |
| 修复前后对比 | 异常机、正常机、修复版本、回读值和复测结果 | 证明修复不是偶然恢复 |

判定口径：IMEI unknown、META 失败、不识卡、radio unavailable 只是表象；没有 assert/NV/产物对比时，不把结论写成最终 root cause。

## 状态升级口径

| 当前状态 | 可做什么 | 升级条件 |
|---|---|---|
| `summarized_with_log_gap` | 用作排查方向和抓证据模板 | 补齐第一坏点证据、对比样本和修复验证后升 `summarized` |
| `summarized` | 用作可复用历史 case | 有正式修复版本或外部确认后可升 `closed` |
| `reference` / `index` | 用作导航或资料入口 | 拆出真实单案后，新建 `doc_type: case` 文件 |

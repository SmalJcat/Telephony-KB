# 常见问题速查流程

这篇用于第一轮分诊。目标是快速判断“坏在哪一层”，然后跳到对应流程、case、代码或配置文档。

## 通用顺序

通用六步、分层判断和最小证据包统一维护在 [[问题定位流程]]。本篇只保留按现象分诊的第一轮检查项。

## 无服务 / Limited Service

| 判断点 | 先看证据 | 下一步 |
|---|---|---|
| SIM 是否 ready | `UiccController`、`SIMRecords`、`dumpsys phone` | 不 ready 转 [SIM识别流程](../20_Service-Flows/SIM/SIM业务流程.md#SIM识别流程) |
| PLMN 是否允许 | forbidden PLMN、manual scan、SIM/漫游限制 | 查 [LTE注册流程](../20_Service-Flows/Network-Registration/LTE注册流程.md) |
| 小区是否合适 | RRC cell selection、barred、suitable、TAC | 看 modem RRC/NAS trace |
| NAS 是否被拒 | Attach Reject / TAU Reject cause | 查 [EMM cause](../10_Basics/CauseCode与缩写.md#LTE-EMM-Reject-常见方向) |
| AP 是否同步错误 | `DATA_REGISTRATION_STATE`、`ServiceStateTracker` | 查 [平台代码与产物速查](../50_Platform-Code/Cross-Platform/平台代码与产物速查.md) |
| 是否 RF/弱场首坏点 | 所有 RAT 搜网失败、RSSI/RSRP 掉底、RACH preamble 打满 | 查 [[无线信号与搜网失败排查]] |

## LTE 注册失败

| 第一坏点 | 典型证据 | 归属 |
|---|---|---|
| RRC 建链失败 | T300 timeout、RACH fail、cell barred | 无线/覆盖/小区配置 |
| Attach / TAU reject | EMM cause 3/6/7/11/12/13/15 | SIM/签约/PLMN/漫游 |
| 默认承载失败 | PDN Connectivity Reject、ESM cause | APN/签约/核心网 |
| modem 成功 AP 无服务 | RIL response 字段、SST 合成状态异常 | AP/RIL 映射 |
| 入口差异 | 开机成功但飞行模式失败，或手动搜网失败 | 平台状态机/scan/select 流程 |
| 全制式搜网失败 | LTE/3G/2G 都失败、RSSI 掉底 | RF/天线/校准/前端器件 |
| reject 后回落失败 | LTE reject 后 3G/2G LU 或 RRC 建链失败 | 分开记录 reject cause 和回落 RAT 第一坏点 |

常见首坏点样例：

| 样例 | 首坏点 | 复用口径 |
|---|---|---|
| [Illegal ME / IMEI 未授权](../40_Case-Library/Registration/Imported_Registration_03_MTN_Uganda售后反馈_无法注册网络.md) | `Attach Reject` / 注册失败原因 `0x06 Illegal ME` | 优先查 IMEI 备案、白名单和可授权 IMEI 对照，不先改 APN/RF |
| [RF 参数/硬件版本错配](../40_Case-Library/Registration/Imported_Registration_01_DAHLIA_450A软件刷在450H手机上_能搜到网络_但是注册不上网络.md) | 已发起 Attach/RRC，但软件与硬件 RF 参数不匹配 | 先核对项目软件、硬件版本、RF 参数包和校准 |
| [注册成功后无 PDP](../40_Case-Library/Registration/Imported_Registration_05_MP6漫游失败.md) | `+EGREG: 5` 已注册漫游，但未发起 PDP | 转 Data / APN 排查，重点看 MVNO/APN 匹配 |
| [SINR 触发频繁重选](../40_Case-Library/Registration/Imported_Registration_07_GH6733B_Symphony反馈信号波动频繁.md) | LRRC 因 SINR 起测异频并重选到弱小区 | 不只看 RSRP，要结合 SINR、频点优先级和重选阈值 |
| [LU/RAU cause 17 后不搜 VPLMN](../40_Case-Library/Registration/Imported_Registration_10_Steering_of_Roaming_Cause17.md) | `SBP_MM_PERFORM_PLMN_SEARCH_AFTER_LU_ABNORMAL = KAL_FALSE` | 查 abnormal counter、NWSEL SBP 和 reject 后 PLMN search |

## 数据无法上网

详细第一轮分诊看 [[数据业务失败排障流程]]。

| 判断点 | 先看证据 | 下一步 |
|---|---|---|
| APN 是否选对 | `DataProfileManager`、APN database、`SETUP_DATA_CALL` | 查 [APN配置](../60_Configuration/APN配置方法_重构.md) |
| PDN reject 属于哪个业务 | `PDN Connectivity Reject`、APN name、ESM cause、PDN purpose | 先区分 default / ims / mms / xcap，再查 [APN分层口径](../60_Configuration/APN配置方法_重构.md#字段配置规则) |
| data call 是否成功 | IP、DNS、ifname、cid | 查 [数据连接与APN流程](../20_Service-Flows/Data/Data业务流程.md#数据连接与APN流程) |
| 只有 IPv6/代理异常 | APN protocol、DNS、TCP/HTTP | 查 Data case |
| 连接成功但无流量 | DNS、TCP 三次握手、路由、VPN/代理 | 走网络抓包和 AP log |
| 系统提示无互联网 | `NetworkMonitor`、`PROBE_DNS`、`PROBE_HTTP/HTTPS`、redirect URL | 区分 DNS timeout、portal 重定向和 HTTP/HTTPS timeout |
| Wi-Fi 切蜂窝异常 | `ConnectivityService` 默认网络切换、data call 重建、validation 状态 | 查 [Wi-Fi切蜂窝案例](../40_Case-Library/Data/2023-10-27_Data_UNISOC_Safaricom_WiFi切蜂窝被重定向.md) |
| 吞吐量差 | CA/DC、band、RSRP/RSRQ/SINR、服务器 | 查 [吞吐量分析流程](../20_Service-Flows/Data/Data业务流程.md#吞吐量分析流程) |
| APN 运行时变更 | `AT+CGDCONT`、`AT+CGACT`、PDN disconnect | 区分 APN 定义、PDN 去激活/激活、detach/attach |
| MMS 失败 | `MMIAPIPDP_Active`、`RequestPdpId`、`MMS_PARSE` | 先确认是否使用了目标运营商的 MMS APN |

常见首坏点样例：

| 样例 | 首坏点 |
|---|---|
| [APN type=xcap 不显示](../40_Case-Library/Data/2022-10-31_Data_UNISOC_APN_XCAP类型被隐藏.md) | APN XML/DB 已有值，但 CarrierConfig 隐藏列表过滤 UI |
| [MMS 大小限制仍为 300KB](../40_Case-Library/Data/2024-11-13_Data_UNISOC_MMS大小限制CarrierConfig.md) | `maxMessageSize` 未按目标运营商覆盖到 600KB |

## IMS 注册失败

IMS 单独分析，不混入 LTE 注册流程。

| 判断点 | 先看证据 | 下一步 |
|---|---|---|
| LTE/NR 数据前置是否正常 | IMS APN、IP、P-CSCF | 查 [IMS注册流程](../20_Service-Flows/IMS/IMS业务流程.md#IMS注册流程) |
| IMS PDN 是否发起 | `PDN_CONNECTIVITY_REQUEST` for IMS、MTK `IMC_REG_CHECK_*` | 无 IMS PDN 先查 [IMS配置方法](../60_Configuration/IMS配置方法.md) |
| SIP REGISTER 是否发出 | REGISTER、401、AKA、200 OK/403/408/503 | 查 IMS case |
| AP 能力是否门控 | CarrierConfig、IMS service capability | 查 [CarrierConfig](../60_Configuration/CarrierConfig配置方法_重构.md) |
| VoWiFi 是否卡在 IKE | `IKE_ATTACH_REQ`、ePDG、unsupported algo | 查 [VoWiFi注册流程](../20_Service-Flows/IMS/IMS业务流程.md#VoWiFi注册流程) |

| 样例 | 首坏点 |
|---|---|

## 通话失败 / 掉话

详细第一轮分诊看 [[通话失败排障流程]]。

| 判断点 | 先看证据 | 下一步 |

| 补充业务失败或回落 | `AT+CCFC(U)`、`AT+ECUSD/EIUSD`、XCAP HTTP、CISS | 查 [补充业务失败排障流程](补充业务失败排障流程.md) |

## SMS / MMS 失败

| 判断点 | 先看证据 | 下一步 |
|---|---|---|
| 短信是否到 RIL | RILJ `SEND_SMS`、`SMSDispatcher`、应用 sentIntent | 无 `SEND_SMS` 先查 AP 侧分支 |
| FDN 是否启用 | FDN list、SMSC、收件人号码 | 查 [SMSC与FDN边界](../60_Configuration/SMS配置方法.md#SMSC来源与FDN边界) |
| modem 是否发出 | CP-DATA/RP-DATA、RP-ACK/RP-ERROR | 到 modem 后再看网络/SMSC |
| MMS APN 是否正确 | `MMIAPIPDP_Active`、`RequestPdpId`、MMSC/proxy | MMS 归 APN/PDP 数据链路排查 |
| 小区广播是否收到 | `APP_MN_SMSCB_MACRO_MSG_IND`、`APP_MN_ETWS_PRIM_IND`、channel range | 区分底层未上报、Mainline 配置限制和 UI 过滤 |

## SIM 不识别

详细第一轮分诊看 [[SIM与运营商名排障流程]]。

| 判断点 | 先看证据 | 下一步 |
|---|---|---|
| 卡槽/上电 | ATR、中断、slot state | 查 [SIM识别流程](../20_Service-Flows/SIM/SIM业务流程.md#SIM识别流程) |
| 应用选择 | USIM/ISIM app state、PIN/PUK | 查 SIM log |
| EF 读取 | IMSI、ICCID、SPN、ECC | 区分卡问题和 AP 解析问题 |
| 单软多硬 | `ro.boot.product.hardware.sku`、slot mapping、NV | singlesim/dualsim 某一 SKU 异常时优先查配置继承 |

## 运营商名称显示错误

| 判断点 | 先看证据 | 下一步 |
|---|---|---|
| SIM 高优先级名称 | EF_SPN、EF_PNN、EF_OPL、ONS | 查 [运营商名称配置方法](../60_Configuration/运营商名称配置方法.md) |
| 网络侧名称 | `s_nitzOperatorInfo`、MM/EMM information、NITZ | 区分 NITZ 是否收到和是否被使用 |
| ROM / XML fallback | `numeric_operator.xml`、`spn-conf`、`UniOperatorNameHandler` | 确认 system/vendor 双来源是否一致 |
| 状态栏最终显示 | `updateSpnDisplay`、`rawPlmn/rawSpn/showPlmn/showSpn` | 不要只看某个配置文件 |
| 飞行模式后变化 | `handleNetworkUnavailable`、radio off、NITZ 缓存 | 查 [NITZ缓存案例](../40_Case-Library/SIM/2025-11-06_SIM_UNISOC_NITZ名称飞行模式后缓存失效.md) |

## Modem 异常 / Radio Restart

| 判断点 | 先看证据 | 下一步 |
|---|---|---|
| 是否 assert | modem dump、subsystem restart、radio unavailable | 查 [Modem稳定性与Assert](../20_Service-Flows/Stability/Modem稳定性与Assert.md) |
| 是否由业务触发 | 触发前正在注册、通话、数据、切换 | 建立业务 case 并链接 Stability case |
| 是否产物问题 | modem image、patch list、编译时间 | 查 [Modem产物与Patch管理](../50_Platform-Code/Cross-Platform/平台代码与产物速查.md#Modem产物与Patch管理) |
| modem 未起 | `MODEM_CTRL:NV_READ`、`boot modem fail`、`MODEM_NOT_ALIVE`、`radio_not_available` | 先确认 fixnv/分区/RFIC/硬件启动链路，再看 SIM/IMEI/ECC 连带现象 |
| RF 初始化失败 | RFIC type 读取失败、`drv_rf_iram.c` assert | 查 RFIC/RFFE 在位、焊接、交叉 RFIC 或单机硬件 |
| AP/BSP 外设诱发 | 通话中 assert，但二分指向 PWM/灯环/外设访问 | 查 [G100 PWM 案例](../40_Case-Library/Stability/Imported_Stability_G100_通话过程中ModemAssert_PWM32K时钟源.md) |

## 无线 / RF 搜网失败

详细第一轮分诊看 [[无线信号与搜网失败排查]]。

| 判断点 | 先看证据 | 下一步 |
|---|---|---|
| 是否所有 RAT 都失败 | LTE/3G/2G 搜网结果、手动搜网列表 | 全失败优先 RF/天线/校准 |
| 小区搜索是否失败 | `CELL_SELECT_CNF`、`PLMN_SEL_FAILURE_IND` | 查 L1/RF 测量 |
| 是否弱场移动 | RSRP/SNR、DSP/L1 时间点、路线记录 | 先补齐 ARM/DSP/AP 时间线 |

# 补充业务失败排障流程

## 阅读入口

这篇用于 Call Forwarding、Call Barring、USSD、UT/XCAP 失败或回落的第一轮分诊。配置字段看 [补充业务配置方法](../60_Configuration/补充业务配置方法.md)，正常流程看 [Call业务流程](../20_Service-Flows/Call/Call业务流程.md#补充业务流程补充资料)。

## 第一轮判断

| 顺序 | 判断点 | 关键证据 | 下一步 |
|---:|---|---|---|
| 1 | 业务类型 | CF/CB/USSD/UT/XCAP/MMI code | 区分设置项、MMI、USSD 和 XCAP |
| 2 | 目标域 | PS/XCAP、IMS USSI、CS/CISS | 不要只看“发生 CSFB” |
| 3 | AP 命令是否发出 | `AT+CCFC(U)`、`AT+ECUSD/EIUSD`、MMI request | AP 未发先查 UI/策略 |
| 4 | XCAP APN 是否正确 | XCAP APN、default APN 复用、PDN reject | 转 APN / 数据排障 |
| 5 | URL / AUID 是否正确 | XCAP `simservs.xml`、HTTP 400/403、AUID 拼接 | 转配置修正 |
| 6 | GBA/BSF/NAF 是否正常 | HTTP 401/200、BSF/NAF、User-Agent | 区分认证过程和最终 XCAP 访问 |
| 7 | CS 回落是否完成 | `EXTENDED_SERVICE_REQUEST`、`CISS__REGISTER/FACILITY/RELEASE_COMPLETE` | 判断 CS 执行结果 |

## USSD 域选

USSD 大部分问题先看选域：运营商是希望走 IMS/USSI，还是走 CS/CISS。历史问题中，运营商定制 USSD code 走 IMS/USSI 后返回 403，对比机路径是回落 CS 并执行 CISS。

关键配置和证据：

| 项 | 说明 |
|---|---|
| `USSI_Enabled` | 是否支持 USSD 走 IMS。`0` 不支持，`1` 支持 |
| `AT+ECUSD` / `AT+EIUSD` | AP 到 modem 的 USSD 请求入口 |
| SIP INVITE dialstring | 证明 USSD 走 IMS/USSI |
| `CISS__REGISTER` / `CISS__FACILITY` | 证明 CS USSD 已执行 |

如果目标运营商只支持 CS USSD，应关闭对应 IMS USSD 能力，让业务直接走 CS。

## XCAP / UT 失败

XCAP 失败不要只看是否 CSFB。常见第一坏点是域选配置、XCAP URL、GBA/BSF/NAF 认证、XCAP APN 或签约。

常见字段：

```text
OPERATOR_NV_MN\mn_apn_info\apn_info_all\apn_info_all[0]\xcap_APN_info\apn_name
OPERATOR_NV_MN\mn_apn_info\apn_info_all\apn_info_all[0]\xcap_APN_info\pdp_type
OPERATOR_NV_MN\mn_apn_info\apn_info_all\apn_info_all[0]\xcap_APN_info\roaming_pdp_type
OPERATOR_NV_IMS\ims_ss_param\ss_param\ss_param[0]\ss_XcapUri
OPERATOR_NV_IMS\ims_ss_param\ss_param\ss_param[0]\ss_XcapPort
OPERATOR_NV_IMS\ims_bsf_param\bsf_param\bsf_param[0]\ss_BsfUri
OPERATOR_NV_IMS\ims_bsf_param\bsf_param\bsf_param[0]\ss_BsfPort
```

历史资料里出现过两类 APN 口径：

```text
APN: A1Si
APN: internet.mnc040.mcc293.gprs
```

这些值只能作为样例，实际配置必须以运营商资料、目标 SIM、对比机和运行时 log 为准。

## PS 补充业务失败

| 现象 | 优先确认 |
|---|---|
| XCAP HTTP 400 | URL/AUID 是否重复拼接，`ss_XcapAuid` 是否硬配错误 |
| XCAP HTTP 403 | 签约、认证、运营商策略、目标 URL |
| 先 401 后 200 | GBA/BSF 认证过程可能正常，不代表后续 XCAP GET 正确 |
| PDN reject | XCAP APN 或复用 default APN 是否符合运营商要求 |
| 失败后 CSFB | 先定位 PS 失败点，再看 CS 回落是否完成 |

## CS 补充业务失败

当前没有独立失败样例沉淀。排查时先按 [通话失败排障流程](通话失败排障流程.md) 判断入口、域选择和 release cause，再看 CS CISS 是否执行到 `REGISTER/FACILITY/RELEASE_COMPLETE`。

## 最小证据包

| 类型 | 内容 |
|---|---|
| AP | MMI/设置入口、AT 命令、IMS/UT/XCAP log、CarrierConfig |
| modem | SIP/XCAP/CISS、域选、PDN/XCAP APN、CSFB/EPSFB 过程 |
| 配置 | 运营商需求、XCAP APN、`ss_domain`、BSF/NAF/XCAP URI/port |
| 对比 | 同卡对比机 log、目标运营商资料、成功与失败路径差异 |

# 补充业务配置方法

## 速查结论

- Call Forwarding / Call Barring / USSD / UT / XCAP 问题先分清走 PS/XCAP、IMS USSI，还是 CS/CISS。
- XCAP 失败后出现 CSFB，不一定是注册或 CSFB 问题；常见第一坏点是域选配置、XCAP URL、GBA/BSF/NAF 认证或 XCAP APN。
- `ss_XcapAuid` 这类 XCAP URL 片段不要按猜测硬配；历史问题中硬配 `simserv.ngn.etsi.org` 会造成 URL 重复拼接，网络返回 400。
- USSD 不是一定走 IMS/USSI；若目标运营商只支持 CS USSD，应关闭对应 IMS USSD 能力，让业务走 CS。
- 失败分析要同时看 AT 命令、域选日志、XCAP HTTP/SIP/CISS 消息和回退触发点。

### 常见失败模式

| 现象 | 优先检查 | 第一坏点判断 |
|---|---|---|
| XCAP HTTP 400 | AUID 是否重复拼接、URL 是否符合运营商要求 | 配置拼接错误早于网络业务失败 |
| USSD 失败 | 运营商是否支持 IMS USSI | 目标域选错时不能先判网络拒绝 |
| CSFB 后才成功 | XCAP/IMS 失败原因和回落触发点 | 要区分主路径失败和 fallback 成功 |

## 域选判断

| 业务 | 常见入口 | PS/XCAP证据 | CS证据 |
|---|---|---|---|
| Call Forwarding | `AT+CCFC` / `AT+CCFCU` / Call settings | XCAP `GET .../simservs.xml`、HTTP 200/400/403 | `EXTENDED_SERVICE_REQUEST`、`CISS__REGISTER` |
| Call Barring | MMI/SSD short code / Call settings | XCAP/UT request、SSDS `path_type=1` | CISS register/facility |
| USSD | `AT+ECUSD` / `AT+EIUSD` | SIP INVITE dialstring、USSI/IMS error 403 | CSFB 后 `CISS__REGISTER` |

关键日志：

```text
SSDS: Domain Decision Done ... path_type=1
ATC_RecNewLineSig ... AT+CCFCU=...
AT_RX ... AT+ECUSD / AT+EIUSD
XCAP GET .../simservs.xml
HTTP/1.1 401 / 200 / 400 / 403
EMM_Extended_Service_Request(service type="MO_CSFB")
CISS__REGISTER / CISS__FACILITY / CISS__RELEASE_COMPLETE
```

## XCAP URL与`ss_XcapAuid`

历史问题中，DUT 访问 XCAP 时拼出了异常 URL：

```text
GET /simserv.ngn.etsi.org/simservs.ngn.etsi.org/users/.../simservs.xml HTTP/1.1
```

对比机正常路径是：

```text
GET https://xcap.../simservs.ngn.etsi.org/users/.../simservs.xml/~~/simservs/communication-diversion
```

根因是 NV 中配置了：

```text
OPERATOR_NV_IMS\ims_ss_param\ss_param\ss_param[0]\ss_XcapAuid="simserv.ngn.etsi.org"
```

导致 XCAP AUID/路径重复拼接。处理原则：

- 先看网络下发和平台默认拼接规则，不要手动补固定域名前缀。
- HTTP 401/200 只说明 GBA/BSF 认证过程可能正常；后续 XCAP GET 返回 400/403 时，要继续看 URL、AUID、rule id、APN 和签约。
- 删除或修正错误 `ss_XcapAuid` 后再验证 CF/CB 是否仍回落 CS。

## USSD域选

历史 USSD 问题中，运营商定制 USSD code 走 IMS/USSI 后返回 403；对比路径是回落 CS 并执行 CISS。修复方向是让该运营商 USSD 走 CS。

常见配置点：

```text
custom_imc_config.c
nvram_ims_profile_ptr->imc_config.ussd_support = 0
```

判断原则：

- 如果运营商不支持 IMS USSD，打开 USSI 只会让业务先走错误域，再触发失败或回退。
- 如果需要 CS USSD，确认 CSFB 后有 `CISS__REGISTER` 和 `CISS__FACILITY`，并最终 `CISS__RELEASE_COMPLETE`。
- 不要只看 UI 提示“失败/回落”，要拆分 IMS USSD 失败点和 CS CISS 执行结果。

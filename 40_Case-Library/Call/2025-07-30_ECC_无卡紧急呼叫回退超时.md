# 无卡紧急呼叫回退超时

## 场景

双卡手机卡 1 插入 SIM，卡 2 无卡；SIM Settings 关闭 `Allow 2G`。本地紧急号码优先走 IMS emergency，但网络不支持 IMS emergency。

## 时间线摘要

| 阶段 | 证据 | 含义 |
|---|---|---|
| 拨号 | ATD192 | AP 发起紧急呼叫 |
| 域选择 | VDM selected IMS | 优先选择 IMS emergency |
| IMS失败 | IMS CC cause 380 | 网络不支持该 IMS emergency |
| CS回退 | `MMCC_EST_REQ`、Extended Service Request | 转 CS domain |
| 找网超时 | `EMM_T_CSFB` 15s 内无 PLMN | CS RAT 找网失败 |
| 上层挂断 | AP timeout / call release | L4 retry 未及时完成 |

## 沉淀规则

紧急呼叫失败要同时记录：

- 号码是否被识别为 emergency number。
- 域选择结果是 IMS 还是 CS。
- IMS 失败 cause。
- CS retry 是否发生，是否受到 2G/3G 开关或 PLMN search 影响。
- 是否触发副卡/无卡 retry。
